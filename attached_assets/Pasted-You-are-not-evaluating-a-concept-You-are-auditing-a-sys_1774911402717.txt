You are auditing a system that is about to be deployed to 10,000 real SME users in Tanzania within 30–90 days.

## 1. REAL-WORLD USAGE SIMULATION (CRITICAL)

Simulate a real Tanzanian duka owner using the system for 7 days:

Day 1:

* Onboards with low digital literacy
* Enters products manually
* Tries first sale

Day 2–3:

* Uses POS during busy hours
* Experiences network drops
* Attempts to track debt customers

Day 4–5:

* Tries to reconcile mobile money
* Notices inconsistencies
* Attempts to generate reports

Day 6–7:

* Trust evaluation phase:

  * Does the owner trust the numbers?
  * Would they continue using it?

OUTPUT:

* Exact moments where trust breaks
* Where confusion occurs
* Where the user abandons the product

## 2. TRUST FAILURE ANALYSIS (MOST IMPORTANT)

In SME software, TRUST = EVERYTHING.

Audit:

* Where could the system produce incorrect data?
* Where could numbers mismatch (sales vs inventory vs cashbook)?
* Where could delays or stale data cause wrong decisions?

Identify:

* Scenarios where the user says:
  "Hii system si sahihi" (This system is not accurate)

Even ONE such moment = churn.
